📊 9-STEP WORKFLOW (All Inside Transaction)
═══════════════════════════════════════════════════════════════════════════════

1. ✅ BEGIN TRANSACTION
   └─ Opens database transaction for atomicity

2. ✅ IDEMPOTENCY CHECK  
   └─ Prevents duplicate orders from retried requests

3. ✅ AUTHENTICATE USER
   └─ Handles both authenticated users and guest checkout

4. ✅ MERGE GUEST CART
   └─ Combines guest cart into user cart if guestId provided

5. ✅ CREATE ORDER
   └─ Creates Order entity with pending status

6. ✅ RESERVE VARIANTS (Critical!)
   └─ Validates stock, reduces available, increases reserved
   └─ Fails entire transaction if out of stock

7. ✅ SAVE INVENTORY TRANSACTIONS
   └─ Records audit trail for each item sold

8. ✅ CLEAR USER CART
   └─ Removes processed items from cart

9. ✅ COMMIT TRANSACTION
   └─ Makes all changes permanent and visible


🔐 SAFETY GUARANTEES
═══════════════════════════════════════════════════════════════════════════════

NO OVERSELLING
  Request: 10 items, Stock: 5
  → Rollback entire transaction, return error
  → No partial orders created

NO DUPLICATE ORDERS
  Request #1: IdempotencyKey="abc" → OrderId=42
  Request #2: IdempotencyKey="abc" → OrderId=42 (SAME!)
  → Network retry safe, only ONE order created

NO PARTIAL ORDERS
  Item 1: Success ✓
  Item 2: Success ✓
  Item 3: Out of Stock ✗
  → ALL changes rollback, clean state
